## Задание 1: Определение типа shell (login/non-login)

### 1.1 Проверка текущей оболочки на тип login shell

Определите, является ли ваша текущая сессия login shell, используя команду `shopt` с опцией `login_shell`:

```bash
shopt -q login_shell && echo "Login shell" || echo "Non-login shell"
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ shopt -q login_shell && echo "Login shell" || echo "Non-login shell"
Non-login shell
```

### 1.2 Проверка через переменную окружения

Проверьте значение переменной `$0`, которая показывает имя оболочки. Если оболочка login, имя начинается с дефиса:

```bash
echo $0
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ echo $0
bash
```

Для login shell вывод был бы:
```sh
[student@lab ~]$ echo $0
-bash
```

---

>Комментарий:
- bash -l — запускает bash с флагом login, читает .bash_profile и т.д., но $0 остаётся bash, а не -bash
- -bash (с дефисом в начале) — это то, что устанавливает сам процесс когда bash запускается как настоящий login shell через execve, где argv[0] передаётся как -bash

shopt login_shell проверяет именно внутренний флаг bash, который выставляется когда argv[0] начинается с -. При bash -l этот флаг не выставляется.

Проверить можно так:
```sh
bash
bash -l -c 'shopt login_shell'  # off
bash --login -c 'shopt login_shell'  # тоже off
```

Чтобы получить настоящий login shell через gnome-terminal нужно:
```sh
gnome-terminal -- bash --login
```

но $0 всё равно будет bash, не -bash. -bash появляется только когда shell запускается через login(1) или getty, где argv[0] выставляется с дефисом наме
ренно.

Итого:
bash -l и bash --login читают login-файлы, но технически не являются login shell с точки зрения shopt login_shell и $0.
