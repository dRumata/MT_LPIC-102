# Практическая работа 02.1. Настройка графических серверов и удалённого доступа в Linux

---

## Цель работы
Научиться определять тип графического сервера, настраивать RDP и VNC серверы на AlmaLinux, а также подключаться к ним с клиентской машины Debian.

---

## Задание 1: Определение типа графического сервера

### 1.1. Проверьте, какой графический сервер используется в системе

Для определения типа графического сервера (X11 или Wayland) проверьте значение переменной окружения `XDG_SESSION_TYPE`:

```sh
echo $XDG_SESSION_TYPE
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ echo $XDG_SESSION_TYPE
wayland
```

или

```sh
[student@lab ~]$ echo $XDG_SESSION_TYPE
x11
```

### 1.2. Получите расширенную информацию о графической сессии

Используйте команду `loginctl` для просмотра детальной информации о текущей сессии:

```sh
loginctl show-session $(loginctl | grep $(whoami) | awk '{print $1}') -p Type
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ loginctl show-session $(loginctl | grep $(whoami) | awk '{print $1}') -p Type
Type=wayland
```

## Задание 2: Просмотр параметров конфигурации графического сервера

### 2.1. Для X11: просмотр конфигурации Xorg

Если в системе используется X11, просмотрите файл конфигурации X-сервера:

```sh
cat /etc/X11/xorg.conf
```

Ожидаемый вывод терминала (при отсутствии файла):
```sh
[student@lab ~]$ cat /etc/X11/xorg.conf
cat: /etc/X11/xorg.conf: No such file or directory
```

Если файл отсутствует, проверьте автоматически сгенерированные конфигурации:

```sh
ls -l /etc/X11/xorg.conf.d/
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ ls -l /etc/X11/xorg.conf.d/
total 4
-rw-r--r--. 1 root root 232 Jan 15 10:23 00-keyboard.conf
```

### 2.2. Для Wayland: проверка параметров compositor

Если используется Wayland, проверьте, какой compositor активен:

```sh
ps aux | grep -E 'wayland|gnome-shell|mutter'
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ ps aux | grep -E 'wayland|gnome-shell|mutter'
student   2145  2.1  3.4 4823456 278432 ?      Sl   09:15   1:23 /usr/bin/gnome-shell
student   2891  0.0  0.0 221904  2304 pts/0    S+   10:12   0:00 grep --color=auto -E wayland|gnome-shell|mutter
```

### 3.4. Настройте firewall для RDP

Откройте порт 3389 в firewall для входящих RDP-подключений:

```sh
sudo firewall-cmd --permanent --add-port=3389/tcp
sudo firewall-cmd --reload
```

Ожидаемый вывод терминала:
```sh

[student@lab ~]$ sudo firewall-cmd --reload
```

## Задание 4: Установка и настройка VNC-сервера на AlmaLinux

> обратите внимание, что для работы VNC-сервера необходимо использовать X11 сессии. Если в системе используется Wayland, рекомендуется переключиться на X11 перед установкой и настройкой VNC-сервера. При осутствии возможности переключиться на X11, рекомендуется использовать RDP для удалённого доступа. В этом случае, выполнение задания 4 может быть пропущено.

### 4.2. Установите пароль VNC для пользователя

Создайте пароль VNC для текущего пользователя:

```sh
vncpasswd
```

Ожидаемый вывод терминала:
```sh

```

### 4.3. Запустите VNC-сервер

Запустите VNC-сервер на дисплее :1:

```sh
vncserver :1
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ vncserver :1

Starting applications specified in /home/student/.vnc/xstartup
Log file is /home/student/.vnc/lab.local:1.log

```

### 4.4. Проверьте запущенные VNC-сессии

Выведите список активных VNC-сессий:

```sh
vncserver -list
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ vncserver -list

:1              5901            4523
```

### 4.5. Настройте firewall для VNC

Откройте порт 5901 (для дисплея :1) в firewall:

```sh
sudo firewall-cmd --permanent --add-port=5901/tcp
sudo firewall-cmd --reload
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo firewall-cmd --permanent --add-port=5901/tcp
success
[student@lab ~]$ sudo firewall-cmd --reload
success
```

### 5.2. Проверьте подключение к RDP-серверу

> Обратите внимание, что адрес 192.168.1.100 является примером. Используйте фактический IP-адрес вашего AlmaLinux сервера.

Проверьте доступность RDP-порта на AlmaLinux сервере:

```sh
nc -zv <IP_адрес_AlmaLinux> 3389
```

Ожидаемый вывод терминала:
```sh
user@debian:~$ nc -zv 192.168.1.100 3389
Connection to 192.168.1.100 3389 port [tcp/ms-wbt-server] succeeded!
```

Запустите Remmina и создайте RDP-подключение через графический интерфейс или выполните команду:

```sh
remmina -c rdp://192.168.1.100
```

### 5.3. Проверьте подключение к VNC-серверу

> Обратите внимание, что vnc-подключение требует указания порта, соответствующего дисплею. Для дисплея :1 это порт 5901. Если на используется wayland, рекомендуется использовать RDP для удалённого доступа, так как VNC не поддерживает Wayland сессии и задание 5.3 может быть пропущено.

Проверьте доступность VNC-порта на AlmaLinux сервере:

```sh
nc -zv <IP_адрес_AlmaLinux> 5901
```

Ожидаемый вывод терминала:
```sh
user@debian:~$ nc -zv 192.168.1.100 5901
Connection to 192.168.1.100 5901 port [tcp/*] succeeded!
```

Запустите Remmina и создайте VNC-подключение:

```sh
remmina -c vnc://192.168.1.100:5901
```
