## Задание 1: Настройка унификации для новых пользователей

### 1.1. Просмотрите содержимое каталога /etc/skel

> Обратите внимание, что для выполнения команд в заданиях 1.1-1.4 необходимо иметь права root или пользоваться sudo.

Каталог `/etc/skel` содержит файлы и каталоги, которые автоматически копируются в домашний каталог каждого нового пользователя.

```sh
ls -la /etc/skel
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# ls -la /etc/skel
total 24
drwxr-xr-x.  2 root root   62 Oct 15 10:23 .
drwxr-xr-x. 78 root root 8192 Jan 15 14:30 ..
-rw-r--r--.  1 root root   18 Aug 10  2021 .bash_logout
-rw-r--r--.  1 root root  141 Aug 10  2021 .bash_profile
-rw-r--r--.  1 root root  492 Aug 10  2021 .bashrc
```

### 1.2. Создайте файл приветствия в каталоге /etc/skel

Создайте файл `.welcome`, который будет копироваться всем новым пользователям.

```sh
echo "Добро пожаловать в систему! Соблюдайте политику безопасности." > /etc/skel/.welcome
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# echo "Добро пожаловать в систему! Соблюдайте политику безопасности." > /etc/skel/.welcome
[root@lab ~]# cat /etc/skel/.welcome
Добро пожаловать в систему! Соблюдайте политику безопасности.
```

### 1.3. Просмотрите текущие настройки в файле /etc/login.defs

Файл `/etc/login.defs` содержит параметры по умолчанию для создания пользователей (UID, срок действия пароля и т.д.).

```sh
grep -E "^UID_MIN|^UID_MAX|^PASS_MAX_DAYS" /etc/login.defs
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# grep -E "^UID_MIN|^UID_MAX|^PASS_MAX_DAYS" /etc/login.defs
UID_MIN                  1000
UID_MAX                 60000
PASS_MAX_DAYS   99999
```

### 1.4. Измените параметр PASS_MAX_DAYS в файле /etc/login.defs

Установите максимальный срок действия пароля в 90 дней для новых пользователей.

```sh
sed -i 's/^PASS_MAX_DAYS.*/PASS_MAX_DAYS   90/' /etc/login.defs
```

Проверьте изменения:

```sh
grep "^PASS_MAX_DAYS" /etc/login.defs
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# grep "^PASS_MAX_DAYS" /etc/login.defs
PASS_MAX_DAYS   90
```

## Задание 2: Создание пользователей

### 2.1. Создайте пользователя "van_gogh"

Используйте команду `useradd` для создания пользователя Ван Гог.

```sh
useradd -c "Vincent van Gogh" van_gogh
```

## Задание 3: Создание групп и распределение пользователей

### 3.1. Создайте группы для классификации художников

Создайте группы: `impressionists` (импрессионисты), `postimpressionists` (постimpрessionисты) и `modernists` (модернисты).

```sh
groupadd impressionists
groupadd postimpressionists
groupadd modernists
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# groupadd impressionists
[root@lab ~]# groupadd postimpressionists
[root@lab ~]# groupadd modernists
[root@lab ~]# tail -3 /etc/group
impressionists:x:1005:
postimpressionists:x:1006:
modernists:x:1007:
```

### 3.2. Добавьте пользователей в соответствующие группы

Распределите художников по группам согласно их художественным направлениям.

```sh
usermod -aG postimpressionists van_gogh
usermod -aG postimpressionists lautrec
usermod -aG postimpressionists cezanne
usermod -aG modernists kahlo
```

Проверьте членство в группах:

```sh
groups van_gogh
groups kahlo
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# groups van_gogh
van_gogh : van_gogh postimpressionists
[root@lab ~]# groups kahlo
kahlo : kahlo modernists
```

## Задание 4: Добавление пользователя в группу wheel

### 4.1. Добавьте пользователя van_gogh в группу wheel

Группа `wheel` традиционно используется для предоставления прав sudo в системах на базе Red Hat.

```sh
usermod -aG wheel van_gogh
```

Проверьте членство:

```sh
groups van_gogh
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# usermod -aG wheel van_gogh
[root@lab ~]# groups van_gogh
van_gogh : van_gogh postimpressionists wheel
```

### 4.2. Проверьте настройки группы wheel в sudoers

Убедитесь, что группе wheel разрешено использование sudo.

```sh
grep "^%wheel" /etc/sudoers
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# grep "^%wheel" /etc/sudoers
%wheel  ALL=(ALL)       ALL
```

### 5.3. Создайте файл конфигурации sudoers для пользователя van_gogh

Создайте файл в каталоге `/etc/sudoers.d/` для настройки ограниченных прав sudo.

```sh
visudo -f /etc/sudoers.d/van_gogh
```

Добавьте следующие строки в файл:

```
# Ограниченные права sudo для van_gogh
van_gogh ALL=(ALL) NOPASSWD: /usr/bin/dnf update, /usr/bin/systemctl restart chronyd, /usr/bin/tail -f /var/log/secure
```

Проверьте синтаксис файла:

```sh
visudo -c -f /etc/sudoers.d/van_gogh
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# visudo -c -f /etc/sudoers.d/van_gogh
/etc/sudoers.d/van_gogh: parsed OK
```

### 5.4. Установите корректные права доступа на файл

```sh
chmod 440 /etc/sudoers.d/van_gogh
```

Ожидаемый вывод терминала:
```sh
[root@lab ~]# chmod 440 /etc/sudoers.d/van_gogh
[root@lab ~]# ls -l /etc/sudoers.d/van_gogh
-r--r-----. 1 root root 156 Jan 15 16:00 /etc/sudoers.d/van_gogh
```

Удалите пользователя van_gogh из группы wheel, чтобы ограничить его права:

```sh
gpasswd -d van_gogh wheel
```
