# Практическая работа 04.1.  Управление systemd-сервисами в Linux

---

## Цель работы
Изучить основы создания и управления systemd-сервисами, научиться настраивать автозапуск служб и анализировать их работу.

---

## Предварительные требования
- ОС: Linux с systemd (Rocky Linux 9 / AlmaLinux 9 / Ubuntu 22.04+)
- Пользователь: student с правами sudo
- Доступ в интернет для загрузки программного обеспечения

## Часть 1: Создание системного сервиса для Forgejo

### Задание 1.1: Создание unit-файла для Forgejo

1. Создайте каталог для данных Forgejo и назначьте права пользователю student.

```sh
sudo mkdir -p /var/lib/forgejo
sudo chown student:student /var/lib/forgejo
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo mkdir -p /var/lib/forgejo
[student@lab ~]$ sudo chown student:student /var/lib/forgejo
[student@lab ~]$
```

2. Создайте systemd unit-файл для сервиса Forgejo с помощью текстового редактора.

```sh
sudo nano /etc/systemd/system/forgejo.service
```

Добавьте следующее содержимое:
```ini
[Unit]
Description=Forgejo Git Service
After=network.target

[Service]
Type=simple
User=student
Group=student
WorkingDirectory=/var/lib/forgejo
ExecStart=/usr/local/bin/forgejo web --config /etc/forgejo/app.ini
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo nano /etc/systemd/system/forgejo.service
[student@lab ~]$
```

### Задание 1.2: Перезагрузка конфигурации systemd

1. Обновите конфигурацию systemd для распознавания нового сервиса.

```sh
sudo systemctl daemon-reload
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo systemctl daemon-reload
[student@lab ~]$
```

2. Проверьте статус созданного сервиса Forgejo.

```sh
sudo systemctl status forgejo.service
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo systemctl status forgejo.service
○ forgejo.service - Forgejo Git Service
     Loaded: loaded (/etc/systemd/system/forgejo.service; disabled; preset: disabled)
     Active: inactive (dead)
```

### Задание 1.3: Запуск и настройка автозапуска сервиса

1. Запустите сервис Forgejo.

```sh
sudo systemctl start forgejo.service
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo systemctl start forgejo.service
[student@lab ~]$
```

2. Настройте автоматический запуск сервиса при загрузке системы.

```sh
sudo systemctl enable forgejo.service
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo systemctl enable forgejo.service
Created symlink /etc/systemd/system/multi-user.target.wants/forgejo.service → /etc/systemd/system/forgejo.service.
```

### Задание 1.4: Проверка доступности сервиса

1. Убедитесь, что сервис запущен и работает корректно.

```sh
sudo systemctl is-active forgejo.service
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo systemctl is-active forgejo.service
active
```

2. Откройте браузер и проверьте доступность Forgejo по адресу `http://localhost:3000`. Вы должны увидеть стартовую страницу Forgejo.

## Часть 2: Создание пользовательского сервиса для VS Code

### Задание 2.1: Создание пользовательского unit-файла

1. Создайте каталог для пользовательских systemd-сервисов.

```sh
mkdir -p ~/.config/systemd/user
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ mkdir -p ~/.config/systemd/user
[student@lab ~]$
```

2. Создайте unit-файл для запуска VS Code Server в пользовательском режиме.

```sh
nano ~/.config/systemd/user/code-server.service
```

Добавьте следующее содержимое:
```ini
[Unit]
Description=VS Code Server for user student
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/code serve-web --host 0.0.0.0 --port 8080 --without-connection-token
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ nano ~/.config/systemd/user/code-server.service
[student@lab ~]$
```

### Задание 2.3: Включение автозапуска пользовательского сервиса

## Часть 4: Оптимизация сервисов

### Задание 4.1: Настройка ограничений ресурсов для Forgejo

1. Откройте unit-файл Forgejo для редактирования.

```sh
sudo nano /etc/systemd/system/forgejo.service
```

Добавьте в секцию `[Service]` ограничения памяти и CPU:
```ini

CPUQuota=50%
```

Ожидаемый вывод терминала:
```sh

[student@lab ~]$
```

2. Перезагрузите конфигурацию и перезапустите сервис для применения изменений.
