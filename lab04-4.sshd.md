# Практическая работа 04.4.  Настройка безопасного SSH-соединения и проброс портов

---

## Цель работы
Научиться настраивать безопасное SSH-соединение между серверами Linux, конфигурировать SSH-клиент и использовать проброс портов для доступа к защищённым сервисам.

---

## Требования
- Сервер: AlmaLinux с установленным SSH-сервером
- Клиент: Debian с установленным SSH-клиентом
- Права пользователя root или sudo для настройки

---

## Часть 1: Установка и базовая проверка SSH

### Задание 1.1: Проверка статуса SSH-сервера на AlmaLinux

1. На сервере AlmaLinux проверьте статус службы SSH. Используйте команду systemctl с аргументом status.

```sh
sudo systemctl status sshd
```

Ожидаемый вывод терминала:
```sh
[student@almalinux ~]$ sudo systemctl status sshd

     Loaded: loaded (/usr/lib/systemd/system/sshd.service; enabled; preset: enabled)
     Active: active (running) since Sun 2026-03-01 10:15:23 UTC; 2h 15min ago
       Docs: man:sshd(8)
             man:sshd_config(5)
   Main PID: 1234 (sshd)
      Tasks: 1 (limit: 11136)
     Memory: 2.5M
```

2. Убедитесь, что SSH-сервер запускается автоматически при загрузке системы.

```sh
sudo systemctl is-enabled sshd
```

Ожидаемый вывод терминала:
```sh
[student@almalinux ~]$ sudo systemctl is-enabled sshd
enabled
```

## Часть 2: Генерация SSH-ключей ed25519

ssh-keygen -t ed25519 -C "student@debian-client"

## Часть 4: Настройка безопасного SSH-сервера

### Задание 4.1: Изменение порта SSH и настройка доступа

1. На сервере AlmaLinux создайте резервную копию конфигурационного файла SSH.

```sh
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup
```

Ожидаемый вывод терминала:
```sh
[student@almalinux ~]$ sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup
[student@almalinux ~]$
```

2. Откройте файл конфигурации SSH для редактирования.

```sh
sudo vi /etc/ssh/sshd_config
```

Найдите и измените следующие параметры (или добавьте их):

```
Port 2222
PermitRootLogin no
PubkeyAuthentication yes
PasswordAuthentication no
AllowUsers student
```

3. Проверьте конфигурацию на наличие синтаксических ошибок.

```sh
sudo sshd -t
```

Ожидаемый вывод терминала (при отсутствии ошибок):
```sh
[student@almalinux ~]$ sudo sshd -t
[student@almalinux ~]$
```

### Задание 4.2: Настройка firewall для нового порта SSH

1. Добавьте правило для порта 2222 в firewall.

```sh
sudo firewall-cmd --permanent --add-port=2222/tcp
```

Ожидаемый вывод терминала:
```sh
[student@almalinux ~]$ sudo firewall-cmd --permanent --add-port=2222/tcp
success
```

2. Перезагрузите конфигурацию firewall и перезапустите SSH-сервер.

```sh
sudo firewall-cmd --reload
sudo systemctl restart sshd
```

Ожидаемый вывод терминала:
```sh
[student@almalinux ~]$ sudo firewall-cmd --reload
success
[student@almalinux ~]$ sudo systemctl restart sshd
[student@almalinux ~]$
```

3. Убедитесь, что SSH-сервер слушает на порту 2222.

```sh
sudo ss -tlnp | grep 2222
```

Ожидаемый вывод терминала:
```sh
[student@almalinux ~]$ sudo ss -tlnp | grep 2222
LISTEN 0      128          0.0.0.0:2222       0.0.0.0:*    users:(("sshd",pid=5678,fd=3))
LISTEN 0      128             [::]:2222          [::]:*    users:(("sshd",pid=5678,fd=4))
```

## Часть 5: Настройка SSH-клиента

### Задание 5.1: Создание файла конфигурации SSH-клиента

1. На клиентской машине Debian создайте или отредактируйте файл конфигурации SSH.

```sh
mkdir -p ~/.ssh
vi ~/.ssh/config
```

Добавьте следующую конфигурацию:

```
Host almaserver
    HostName 192.168.1.100
    Port 2222
    User student
    IdentityFile ~/.ssh/id_ed25519
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

2. Установите правильные права доступа на файл конфигурации.

```sh
chmod 600 ~/.ssh/config
```

Ожидаемый вывод терминала:
```sh
[student@debian ~]$ chmod 600 ~/.ssh/config
[student@debian ~]$
```

3. Проверьте подключение к серверу, используя созданный алиас.

```sh
ssh almaserver
```

Ожидаемый вывод терминала:
```sh
[student@debian ~]$ ssh almaserver
Last login: Sun Mar  1 12:30:45 2026 from 192.168.1.50
[student@almalinux ~]$
```

## Часть 7: Настройка проброса портов через SSH

### Задание 7.1: Использование локального проброса портов (-L)

1. На клиентской машине Debian установите SSH-туннель с пробросом локального порта 8080 на удалённый порт 80 сервера. Используйте опцию -L.

```sh
ssh -L 8080:localhost:80 almaserver
```

Ожидаемый вывод терминала:
```sh
[student@debian ~]$ ssh -L 8080:localhost:80 almaserver
Last login: Sun Mar  1 13:15:32 2026 from 192.168.1.50
[student@almalinux ~]$
```

2. Не закрывая SSH-соединение, откройте новый терминал на клиенте Debian и проверьте доступ к веб-серверу через туннель.

```sh
curl http://localhost:8080
```

Ожидаемый вывод терминала:
```sh
[student@debian ~]$ curl http://localhost:8080
<h1>Welcome to AlmaLinux Web Server</h1><p>SSH Port Forwarding Test</p>
```

### Задание 7.2: Проброс портов в фоновом режиме

1. Закройте предыдущее SSH-соединение (Ctrl+D) и создайте туннель в фоновом режиме с опциями -f (фон) и -N (без выполнения команд).

```sh
ssh -f -N -L 8080:localhost:80 almaserver
```

Ожидаемый вывод терминала:
```sh
[student@debian ~]$ ssh -f -N -L 8080:localhost:80 almaserver
[student@debian ~]$
```

2. Проверьте, что процесс SSH-туннеля запущен.

```sh
ps aux | grep "ssh.*8080"
```

Ожидаемый вывод терминала:
```sh
[student@debian ~]$ ps aux | grep "ssh.*8080"
student   7890  0.0  0.1  12345  6789 ?        Ss   13:25   0:00 ssh -f -N -L 8080:localhost:80 almaserver
student   7925  0.0  0.0   6789  2345 pts/0    S+   13:26   0:00 grep ssh.*8080
```

3. Откройте веб-браузер на клиенте и перейдите по адресу http://localhost:8080 для проверки доступа к веб-серверу через SSH-туннель.
