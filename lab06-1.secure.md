## Задание 1. Поиск файлов с установленным битом SUID

**Описание:** Бит SUID (Set User ID) позволяет пользователю запускать исполняемый файл с правами владельца файла. Это потенциально опасно, поэтому важно знать, какие файлы в системе имеют этот бит.

1. Найдите все файлы в системе с установленным битом SUID. Используйте команду find с ключом `-perm /4000`.

```sh
sudo find / -type f -perm /4000 2>/dev/null
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo find / -type f -perm /4000 2>/dev/null
/usr/bin/sudo
/usr/bin/su
/usr/bin/passwd
/usr/bin/mount
/usr/bin/umount
/usr/bin/chsh
/usr/bin/chfn
/usr/libexec/dbus-daemon-launch-helper
```

2. Найдите файлы с установленным битом SUID в директории `/usr/bin` с подробной информацией.

```sh
sudo find /usr/bin -type f -perm /4000 -ls 2>/dev/null
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo find /usr/bin -type f -perm /4000 -ls 2>/dev/null
  1234567  156 -rwsr-xr-x   1 root     root       159344 Jan 15  2025 /usr/bin/sudo
  1234568   64 -rwsr-xr-x   1 root     root        65448 Jan 15  2025 /usr/bin/su
  1234569   40 -rwsr-xr-x   1 root     root        39632 Jan 15  2025 /usr/bin/passwd
  1234570   48 -rwsr-xr-x   1 root     root        48976 Jan 15  2025 /usr/bin/mount
```

## Задание 2. Поиск файлов с установленным битом SGID

**Описание:** Бит SGID (Set Group ID) позволяет запускать файл с правами группы-владельца. Для директорий SGID обеспечивает наследование группы для новых файлов.

1. Найдите все файлы и директории с установленным битом SGID в системе.

```sh
sudo find / -type f -perm /2000 2>/dev/null
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo find / -type f -perm /2000 2>/dev/null
/usr/bin/write
/usr/bin/wall
/usr/sbin/unix_chkpwd
/usr/sbin/netreport
/usr/libexec/openssh/ssh-keysign
```

2. Найдите директории с установленным битом SGID в `/home`.

```sh
sudo find /home -type d -perm /2000 2>/dev/null
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo find /home -type d -perm /2000 2>/dev/null
/home/shared
/home/projects/team
```

## Задание 3. Проверка статуса паролей пользователей

**Описание:** Команда `passwd -S` позволяет администратору проверить статус пароля пользователя без его изменения.

1. Проверьте статус пароля пользователя van_gogh.

```sh
sudo passwd -S van_gogh
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo passwd -S van_gogh
van_gogh PS 2025-02-15 0 99999 7 -1 (Password set, SHA512 crypt.)
```

**Расшифровка вывода:**
- `PS` — пароль установлен (Password Set)
- `2025-02-15` — дата последнего изменения пароля
- `0` — минимальное количество дней между изменениями
- `99999` — максимальное количество дней до истечения срока действия
- `7` — количество дней предупреждения перед истечением
- `-1` — количество дней до блокировки после истечения

2. Проверьте статусы паролей для нескольких пользователей одновременно.

```sh
for user in lautrec cezanne kahlo; do sudo passwd -S $user; done
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ for user in lautrec cezanne kahlo; do sudo passwd -S $user; done
lautrec PS 2025-02-20 0 99999 7 -1 (Password set, SHA512 crypt.)
cezanne LK 2025-01-10 0 99999 7 -1 (Password locked.)
kahlo NP 2025-02-01 0 99999 7 -1 (Empty password.)
```

**Возможные статусы:**
- `PS` — пароль установлен
- `LK` — учетная запись заблокирована
- `NP` — пароль не установлен

## Задание 4. Блокировка учетной записи пользователя

**Описание:** Команда `usermod -L` блокирует учетную запись пользователя, добавляя восклицательный знак перед хешем пароля в `/etc/shadow`.

1. Заблокируйте учетную запись пользователя lautrec.

```sh
sudo usermod -L lautrec
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo usermod -L lautrec
[student@lab ~]$
```

2. Проверьте статус заблокированной учетной записи.

```sh
sudo passwd -S lautrec
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo passwd -S lautrec
lautrec LK 2025-02-20 0 99999 7 -1 (Password locked.)
```


## Задание 9. Определение процессов, использующих файл или директорию

**Описание:** Команда `fuser` идентифицирует процессы, которые используют указанные файлы или файловые системы.

1. Определите, какие процессы используют директорию `/home/van_gogh`.

```sh
sudo fuser -v /home/van_gogh
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo fuser -v /home/van_gogh
                     USER        PID ACCESS COMMAND
/home/van_gogh:      van_gogh   3456 ..c.. bash
                     van_gogh   3789 ..c.. vim
```

**Расшифровка ACCESS:**
- `c` — текущая директория (current directory)
- `e` — исполняемый файл
- `r` — корневая директория
- `f` — открытый файл

2. Определите процессы, использующие конкретный файл, и завершите их (будьте осторожны!).

```sh
sudo fuser -v /home/kahlo/document.txt
```

Ожидаемый вывод терминала:
```sh
[student@lab ~]$ sudo fuser -v /home/kahlo/document.txt
                     USER        PID ACCESS COMMAND
/home/kahlo/document.txt:
                     kahlo      4521 f.... vim
```

Для завершения процесса можно использовать:
```sh
sudo fuser -k /home/kahlo/document.txt
```
