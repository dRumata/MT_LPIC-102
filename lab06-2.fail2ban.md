# Практическая работа 06.2.  Настройка fail2ban для защиты SSH

---

## Цель работы
Научиться устанавливать и настраивать систему fail2ban для автоматической блокировки IP-адресов при попытках несанкционированного доступа по SSH.

---

## Описание лабораторной среды
- **Сервер**: almaserver (AlmaLinux)
- **Клиент**: debian
- **SSH порт на сервере**: 2222
- **Учетные записи**: student (правильный логин), admin (для тестирования блокировки)
- **Аутентификация**: SSH-ключ для пользователя student

## Задание 2: Настройка fail2ban для SSH

1. Создайте локальный конфигурационный файл для jail SSH. Используйте текстовый редактор для создания файла `/etc/fail2ban/jail.local`.

```sh
sudo vi /etc/fail2ban/jail.local
```

Добавьте следующее содержимое в файл:
```ini
[DEFAULT]
bantime = 10m
findtime = 10m
maxretry = 3

[sshd]
enabled = true
port = 2222
logpath = /var/log/secure
backend = systemd
```

2. Проверьте корректность конфигурации fail2ban с помощью команды fail2ban-client.

```sh
sudo fail2ban-client -d
```

Ожидаемый вывод терминала (фрагмент):
```sh
[student@almaserver ~]$ sudo fail2ban-client -d
['set', 'loglevel', 'DEBUG']
['start']
...
Server ready
```

## Задание 5: Разблокировка IP-адреса

1. Разблокируйте IP-адрес клиента вручную с помощью команды fail2ban-client. Замените `192.168.1.100` на фактический IP-адрес вашего клиента.

```sh
sudo fail2ban-client set sshd unbanip 192.168.1.100
```

Ожидаемый вывод терминала:
